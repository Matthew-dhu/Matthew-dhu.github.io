1 SpringCloud简介

1.1 简介

SpringCloud是Spring旗下的项目之一，它将现在非常流行的一些技术整合到一起，实现了诸如：配置管理，服务发现，智能路由，负载均衡，熔断器，控制总线，集群状态等等功能。其主要涉及的组件包括：

  • Eureka：注册中心
  • Zuul：服务网关
  • Ribbon：负载均衡
  • Feign：服务调用
  • Hystix：熔断器

1.2 SpringCloud版本

​ SpringCloud版本号不是我们熟悉的数字，而是使用英文代号。下表是各版本及其依赖的SpringBoot版本的对应。

SpringCloud SpringBoot
Angle 1.2.x
Brixton 1.3.x
Camden 1.4.x
Dalston 1.5.x
Edgware 1.5.x
Finchley 2.x

搭建环境时尽量选择对应版本的SpringBoot，避免一些不必要的错误。

2 SpringCloud的使用

下面将通过一个实例来展示SpringCloud的基本用法。

目标：一个工程下两个模块，一个提供服务（server）一个使用服务（consumer）完成模块（服务）间的相互调用。

2.1创建主工程

创建一个maven工程作为父工程，引入pom依赖，做好版本控制。代码如下。

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.marui</groupId>
    <artifactId>SpringCloudDemo</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>pom</packaging>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.0.3.RELEASE</version>
        <relativePath/>
    </parent>

    <properties>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <project.reporting.outputEncoding>UTF-8</project.reporting.outputEncoding>
        <java.version>1.8</java.version>
        <spring-cloud.version>Finchley.RELEASE</spring-cloud.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.cloud</groupId>
                <artifactId>spring-cloud-dependencies</artifactId>
                <version>${spring-cloud.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>

2.2 创建erueka服务器

2.2.1 了解eureka

首先，从一个生活中的例子来了解erueka是用来做什么的。

网约车出现以前，人们出门叫车只能叫出租车。一些私家车想做出租却没有资格，被称为黑车。而很多人想要约车，但是无奈出租车太少，不方便。私家车很多却不敢拦，而且满大街的车，谁知道哪个才是愿意载人的。一个想要，一个愿意给，但是缺少引子，缺乏管理。

此时滴滴这样的网约车平台出现了，所有想载客的私家车全部到滴滴注册，记录你的车型（服务类型），身份信息（联系方式）。这样提供服务的私家车，在滴滴那里都能找到，一目了然。

此时要叫车的人，只需要打开APP，输入你的目的地，选择车型（服务类型），滴滴自动安排一个符合需求的车到你面前，为你服务。

Eureka就好比是滴滴，负责管理、记录服务提供者的信息。服务调用者无需自己寻找服务，而是把自己的需求告诉Eureka，然后Eureka会把符合你需求的服务告诉你。

同时，服务提供方与Eureka之间通过“心跳”机制进行监控，当某个服务提供方出现问题，Eureka自然会把它从服务列表中剔除。

这就实现了服务的自动注册、发现、状态监控。

所以完成模块间的相互调用我们使用erueka而不是直接调用。首先创建eureka服务器

2.2.1 使用eureka

用Spring Initializr 创建一个新的springboot子模块

添加eureka server

创建之后pom依赖以及引好。

在主启动类上添加@EnableEurekaServer注解。

@SpringBootApplication
@EnableEurekaServer
public class EruekaApplication {
    public static void main(String[] args) {
        SpringApplication.run(EruekaApplication.class, args);
    }
}

修改配置，其中spring: application: name: 是指在eureka注册中心里该服务的名字，eureka: client: service-url: 写的是注册中心的地址。

server:
  port: 10086
spring:
  application:
    name: eureka
eureka:
  client:
    service-url:
      defaultZone: http://127.0.0.1:10086/eureka #注册到eureka（eureka本身也需要注册）

此时我们就可以打开10086端口看到eureka的界面，看到已注册的服务（目前只有eureka一个）。

2.3创建服务提供者 server

创建过程与上一步类似，不在赘述。但是在主启动类上不再使用@EnableEurekaServer注解，而是@EnableDiscoveryClient注解。

创建一个controller，需要注意的是，springboot默认只会帮我们扫描主启动类下的包，所以controller最好放在与主启动类同级目录下。

@RestController
public class TestController {
    @RequestMapping("/hello")
    public String hello(){
        return "hello! I am from 8081";
    }
}

配置信息同上，我们server使用的是8081端口

server:
  port: 8081
spring:
  application:
    name: server
eureka:
  client:
    service-url:
      defaultZone: http://127.0.0.1:10086/eureka

2.4 创建服务消费者consumer

与server相同，但是在启动类下我们需要注入RestTemplate，用于Http调用。

@Bean
RestTemplate restTemplate() {
    return new RestTemplate();
}

编写controller

@RestController
public class TestController {
    @Autowired
    private RestTemplate restTemplate;
    @Autowired
    private DiscoveryClient discoveryClient;

    @RequestMapping("consume")
    public String consume(){
        List<ServiceInstance> instances = discoveryClient.getInstances("server");
        // 获取服务实例，因为只有一个,因此我们直接get(0)获取
        ServiceInstance instance = instances.get(0);
        // 获取ip和端口信息
        String url = "http://"+instance.getHost() + ":" + instance.getPort()+"/hello";
        //调用服务
        return restTemplate.getForObject(url,String.class);
    }
}
